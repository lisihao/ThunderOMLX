## 3. 专家评审问题

在可行性验证前，**审判官**（deepseek-r1）和**稳健派**（gemini-2.5-pro）发现以下 CRITICAL 缺陷：

### 3.1 位置依赖性问题 (CRITICAL)

**问题**: KV Cache 是位置依赖的。全局 `_sub_block_index` 允许任意位置的子块匹配，导致：

```python
# ❌ 错误设计
_sub_block_index: Dict[bytes, SubBlockCacheEntry]  # 全局索引

# 问题：从位置 [256-287] 提取的 KV cache 被用于位置 [512-543]
# 导致：Attention 计算错误，模型输出损坏
```

**影响**: 模型输出不正确，缓存完全失效。

**修复方案**: 父作用域子块索引（已在方案中列出，但增加复杂度）

### 3.2 孤儿子块问题 (HIGH)

**问题**: 主 block 被驱逐时，子 block 成为孤儿：

```python
# 驱逐主 block M
del _prefix_index[main_block_hash]

# 问题：_sub_block_index 中的子 block S 仍然存在
# 新请求匹配到 S → 尝试访问 M → KeyError 崩溃
```

**影响**: 程序崩溃或内存泄漏。

**修复方案**: 级联删除（需实现复杂的父子引用计数）
