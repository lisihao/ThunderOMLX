         ▼                  ▼                  ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│ContentDetector  │  │BoundaryExtractor│  │DynamicChunker   │
│(类型检测)       │  │(边界提取)       │  │(动态分块)       │

### 1. 内容类型检测 (`content_detector.py`)

**功能**: 自动识别文本类型，适配不同的分块策略

| 类型 | 检测条件 | 策略调整 |
|------|----------|----------|
| **DIALOGUE** | ≥2 个对话标记 (`User:`/`Assistant:`) | min_size=512 (允许短对话) |
| **DOCUMENT** | ≥3 段落，平均长度 >100 字符 | max_size=6144 (允许长段落) |
| **CODE** | ≥2 个代码特征 (`def`/`class`/`function`) | min_size=256 (允许短函数) |
| **MIXED** | 满足 2+ 类型 | 提取所有类型边界 |
| **GENERIC** | 默认 | 使用默认参数 |

### 2. 语义边界提取 (`boundary_extractor.py`)

**功能**: 识别多层次语义边界

| 边界类型 | 模式 | 强度 |
|----------|------|------|
| **DIALOGUE** | `User:`, `Assistant:`, `<\|im_start\|>` | 1.0 (强) |
| **PARAGRAPH** | `\n\n` | 1.0 (强) |
| **CODE_BLOCK** | ` ``` `, `function`, `class` | 1.0 (强) |
| **SENTENCE** | `. ! ?` + 空格 | 0.5 (中) |

**关键机制**:
- 文本 offset → token offset 映射（近似）
- 去重：同位置只保留强度最高的边界
- 排序：按 token_offset 排序

### 3. 动态分块算法 (`dynamic_chunker.py`)

**核心算法**: **Greedy Boundary-Aware Packing**

```python
# 弹性范围
target_size = 4096
flexibility = 0.125  # ±12.5%
min_size = 3584  # 4096 - 512
max_size = 4608  # 4096 + 512

# 评分函数
def score(boundary):
    strength_score = boundary.strength * 1000  # 强度权重
    distance_penalty = abs(boundary.token_offset - ideal)
    return strength_score - distance_penalty

# 贪心选择：在弹性范围内选择得分最高的边界
best_boundary = max(candidates, key=score)
```

**优势**:
- 平衡语义完整性和 size 利用率
- 强边界优先（对话/段落/代码块）
- 距离惩罚确保不偏离 target 太远

### 4. 质量验证 (`quality_validator.py`)

**三大质量指标**:

| 指标 | 定义 | 目标 |
|------|------|------|
| **boundary_integrity** | 在强边界处切分的比例 | ≥95% |
| **size_uniformity** | 1.0 - CV (变异系数) | ≥85% (CV <0.15) |
| **cross_boundary_rate** | 在边界附近但不在边界处切分的比例 | ≤5% |

**综合得分**:
```
overall_score = boundary_integrity * 0.5 +
                size_uniformity * 0.3 +
                (1 - cross_boundary_rate) * 0.2
```

**自动回退**: 如果 `overall_score < 0.8`，自动回退到固定分块

### 5. 主编排器 (`intelligent_chunker.py`)

**完整流程**:
```
1. ContentDetector.detect(text) → ContentType
2. BoundaryExtractor.extract(text, tokens, content_type) → Boundaries
3. DynamicChunker.chunk(tokens, boundaries, content_type) → Chunks
4. QualityValidator.validate(chunks, boundaries) → ChunkQuality
5. if quality.overall_score < 0.8:
       回退到固定分块
6. 执行分块 prefill
```

**便捷 API**:
```python
from omlx.chunking import intelligent_chunked_prefill

output, stats, quality = intelligent_chunked_prefill(
    model=model,
    tokenizer=tokenizer,
    prompt=prompt,
    target_size=4096,
    flexibility=0.125,
    max_tokens=100,
    verbose=True
)
```

## 测试结果

### 测试场景

| 场景 | Tokens | 内容类型 | Chunk 数量 |
|------|--------|----------|-----------|
| 对话格式 | 15,817 | DIALOGUE | 4 |
| 文档格式 | 16,192 | DOCUMENT | 4 |
| 代码格式 | 15,633 | MIXED | 4 |

### 质量指标

| 场景 | 边界完整性 | Size 均匀性 | 跨边界率 | 综合得分 |
|------|-----------|------------|---------|---------|
| **对话格式** | 100.00% | 66.06% | 0.00% | **88.69%** ✅ |
| **文档格式** | 100.00% | 49.46% | 0.00% | **82.82%** ✅ |
| **代码格式** | 100.00% | 91.82% | 0.00% | **97.55%** 🏆 |

**关键发现**:
- ✅ **所有测试都超过 80% 最低质量标准**
- 🏆 **代码格式得分最高 (97.55%)**，因为代码块边界清晰
- ⚠️ 文档格式 size 均匀性较低 (49.46%)，因为段落长度差异大（符合预期）
- ✅ **100% 边界完整性**：所有切分都在强边界处
- ✅ **0% 跨边界率**：没有破坏语义边界

### 性能对比

| 场景 | 固定分块吞吐量 | 智能分块吞吐量 | 性能开销 |
|------|---------------|---------------|---------|
| 对话格式 | 750.8 tok/s | 675.4 tok/s | **+11.3%** ⚠️ |
| 文档格式 | 744.0 tok/s | 728.4 tok/s | **+2.2%** ✅ |
| 代码格式 | 728.8 tok/s | 718.5 tok/s | **+1.4%** ✅ |

**关键发现**:
- ✅ **平均性能开销 <5%**（除对话格式外）
- ✅ 质量提升（语义完整性）vs 轻微性能损失（<5%）是值得的权衡
- ⚠️ 对话格式开销较高 (+11.3%)，可能因为边界检测复杂度更高

## 核心优势

### vs LangChain

| 特性 | LangChain | ThunderOMLX 智能分块 |
|------|-----------|---------------------|
| 边界识别 | 简单分隔符 | 多层次语义边界（对话/段落/代码块/句子） |
| Chunk Size | 固定或基于字符数 | 动态 (512-6K, target 4K ±12.5%) |
| 内容自适应 | ❌ | ✅ (5 种内容类型) |
| 质量验证 | ❌ | ✅ (3 个指标，自动回退) |
| 评分函数 | ❌ | ✅ (强度 * 1000 - 距离) |

### vs LlamaIndex

| 特性 | LlamaIndex | ThunderOMLX 智能分块 |
|------|-----------|---------------------|
| Chunk Strategy | Sentence/Semantic | Greedy Boundary-Aware Packing |
| 语义边界 | 依赖 NLP 库 | 正则表达式（零依赖，快速） |
| Token Mapping | 精确（需 tokenizer offset） | 近似（chars/4，快速） |
| 质量保证 | ❌ | ✅ (质量指标 + 自动回退) |
| 代码块识别 | ❌ | ✅ (代码格式特化) |
