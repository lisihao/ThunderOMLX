## 2. 优化设计

### 2.1 两级匹配策略

```
┌─────────────────────────────────────────────────────────────┐
│                   _messages_equal()                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─ Fast Path (O(1)) ────────────────────────────────────┐ │
│  │  if content1 == content2 and role1 == role2:           │ │
│  │      return True  # exact match, 最快                   │ │
│  └────────────────────────────────────────────────────────┘ │
│                        ↓ 失败                                │
│  ┌─ Fallback (O(n)) ─────────────────────────────────────┐ │
│  │  if role1 != role2:                                    │ │
│  │      return False  # role 必须相同                      │ │
│  │                                                         │ │
│  │  similarity = _compute_similarity(content1, content2)  │ │
│  │  return similarity >= fuzzy_threshold  # 相似度检测     │ │
│  └────────────────────────────────────────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**关键点**:
- Fast path 覆盖 90%+ 场景（大多数消息完全相同）
- Fallback 处理边缘情况（标点、空格差异）
- Role 必须 exact match（system/user/assistant 不能混淆）

### 2.2 相似度算法选择

| 算法 | 时间复杂度 | 内存 | 特点 | 适用场景 |
|------|-----------|------|------|----------|
| **Levenshtein Distance** | O(m*n) | O(min(m,n)) | 经典编辑距离 | ✅ 推荐（标准） |
| Jaro-Winkler | O(m+n) | O(1) | 对前缀敏感 | ⚠️ 不适合（后缀变化） |
| Cosine Similarity | O(m+n) | O(m+n) | 基于词向量 | ❌ 过重（需要分词） |
| Hamming Distance | O(n) | O(1) | 仅等长字符串 | ❌ 不适用 |

**选择**: **Levenshtein Distance** (标准化版本)

### 2.3 相似度计算

```python
def _compute_similarity(s1: str, s2: str) -> float:
    """
    计算两个字符串的相似度 (0.0 - 1.0)

    使用标准化的 Levenshtein 距离:
    similarity = 1 - (edit_distance / max_len)

    Args:
        s1: 字符串 1
        s2: 字符串 2

    Returns:
        相似度分数 (0.0 = 完全不同, 1.0 = 完全相同)
    """
    if s1 == s2:
        return 1.0  # fast path

    if not s1 or not s2:
        return 0.0  # 空字符串

    # Levenshtein distance
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    # 初始化
    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j

    # 动态规划
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            cost = 0 if s1[i-1] == s2[j-1] else 1
            dp[i][j] = min(
                dp[i-1][j] + 1,      # 删除
                dp[i][j-1] + 1,      # 插入
                dp[i-1][j-1] + cost  # 替换
            )

    edit_distance = dp[m][n]
    max_len = max(m, n)
    similarity = 1.0 - (edit_distance / max_len)

    return similarity
```

**优化**: 使用 O(min(m,n)) 空间（滚动数组）

**修改**: `adapter.py:348-355`

```python
def _messages_equal(
    self,
    m1: Dict[str, Any],
    m2: Dict[str, Any],
    fuzzy: bool = True  # 新增参数
) -> bool:
    """检查两个消息是否相等（支持 fuzzy match）"""
    content1 = self._extract_content(m1)
    content2 = self._extract_content(m2)
    role1 = self._extract_role(m1)
    role2 = self._extract_role(m2)

    # Role 必须 exact match
    if role1 != role2:
        return False

    # Fast path: exact match
    if content1 == content2:
        return True

    # Fallback: fuzzy match (if enabled)
    if fuzzy and self.fuzzy_match_enabled:
        similarity = self._compute_similarity(content1, content2)
        logger.debug(
            f"Fuzzy match: similarity={similarity:.3f}, "
            f"threshold={self.fuzzy_threshold}"
        )
        return similarity >= self.fuzzy_threshold

    return False
```

### 3.4 Phase 4: 性能优化 (15 分钟)

**优化 1**: 滚动数组降低空间复杂度

```python
def _compute_similarity_optimized(self, s1: str, s2: str) -> float:
    """优化版：O(min(m,n)) 空间"""
    if s1 == s2:
        return 1.0

    if not s1 or not s2:
        return 0.0

    # 确保 s1 是较短的字符串
    if len(s1) > len(s2):
        s1, s2 = s2, s1

    m, n = len(s1), len(s2)
    prev = list(range(n + 1))
    curr = [0] * (n + 1)

    for i in range(1, m + 1):
        curr[0] = i
        for j in range(1, n + 1):
            cost = 0 if s1[i-1] == s2[j-1] else 1
            curr[j] = min(
                prev[j] + 1,      # 删除
                curr[j-1] + 1,    # 插入
                prev[j-1] + cost  # 替换
            )
        prev, curr = curr, prev

    edit_distance = prev[n]
    similarity = 1.0 - (edit_distance / n)

    return similarity
```

**优化 2**: 长消息截断（避免过长字符串）

self._similarity_cache: Dict[Tuple[str, str], float] = {}

def _compute_similarity_cached(self, s1, s2):
    key = (s1[:100], s2[:100])  # 只缓存前 100 字符的 hash
    if key in self._similarity_cache:
